Le data-question dans le HTML est un attribut personnalisé (aussi appelé data attribute) qui permet d'associer chaque verset à une question spécifique.

🔹 Pourquoi l'utiliser ?

Dans ton code, chaque verset est relié à une question. Quand on clique sur un verset, on a besoin de savoir :

  • À quelle question il est lié
  • Si cette question est celle qui est actuellement affichée

Le data-question permet justement de stocker cette information dans le HTML et de l'utiliser en JavaScript.

🔹 Comment ça marche ?

Dans le HTML, chaque verset a un data-question qui indique à quelle question il appartient :

html
<div class="verse verse1" data-question="1">
    <div class="ref"><h1>Marc 5:14</h1></div>
    <div class="text"><p>Lorem ipsum dolor sit amet...</p></div>
</div>

<div class="verse verse2" data-question="2">
    <div class="ref"><h1>Jean 3:16</h1></div>
    <div class="text"><p>Autre texte pour la deuxième question...</p></div>
</div>

Dans le JavaScript, on récupère cette valeur avec :

javascript
const verseQuestion = parseInt(this.getAttribute("data-question"));

Et on compare avec la question actuelle (currentIndex + 1) :

javascript
if (verseQuestion === currentIndex + 1) {
    this.style.backgroundColor = "green"; // Verset lié → Vert
} else {
    this.style.backgroundColor = "red"; // Autre verset → Rouge
}

🔹 Pourquoi ne pas juste utiliser id ou class ?

  • id : Il doit être unique, donc on ne peut pas réutiliser le même ID pour plusieurs éléments.
  • class : Une classe sert plutôt au style et à la mise en page. Ici, on a besoin de stocker une information spécifique sur chaque verset.

Le data attribute (data-question) est donc la meilleure solution pour stocker ce type d'information sans interférer avec le CSS.

✅ Avantages des data-attributes

✔ Permet de lier dynamiquement des éléments sans écrire plein de if dans le code.
✔ Garde le HTML propre et structuré.
✔ Facilite les mises à jour : si tu ajoutes un nouveau verset, il suffit de lui donner le bon data-question, et ça marchera sans changer le JavaScript.

📌 Exemple pratique :

Imaginons qu'on ait une nouvelle question Question 3 et un nouveau verset lié :

html
<div class="question question3" id="question3">
    <h1>Nouvelle question ici...</h1>
</div>

<div class="verse verse4" data-question="3">
    <div class="ref"><h1>Mathieu 6:33</h1></div>
    <div class="text"><p>Un verset pour la question 3...</p></div>
</div>

Pas besoin de changer le JavaScript, ça marchera automatiquement ! 🚀

📌 parseInt() : Ça veut dire quoi ?

parseInt() est une fonction JavaScript qui convertit une chaîne de caractères (string) en nombre entier (integer).

🔹 Pourquoi l'utiliser ici ?

Dans ton code, data-question est stocké sous forme de texte dans le HTML, même si c'est un nombre :

html
<div class="verse" data-question="1">...</div>

Quand on récupère la valeur avec getAttribute(), on obtient une chaîne de caractères "1", pas un nombre !
Donc, on utilise parseInt() pour la convertir en nombre entier :

javascript
const verseQuestion = parseInt(this.getAttribute("data-question"));

🔹 Exemple simple

javascript
const valeurTexte = "42";
const valeurNombre = parseInt(valeurTexte);

console.log(typeof valeurTexte);  // string
console.log(typeof valeurNombre); // number
console.log(valeurNombre + 10);   // 52 (car c'est bien un nombre)

Si on n'utilise pas parseInt(), JavaScript va concaténer au lieu d'additionner :

javascript
console.log("42" + 10); // "4210" (c'est une chaîne)

🔹 Pourquoi parseInt() est important ?

  1. Éviter des erreurs → Si on compare "1" === 1, ça donne false, alors que parseInt("1") === 1 donne true.
  2. Pouvoir faire des calculs → On ne peut pas additionner "10" + 5, mais parseInt("10") + 5 fonctionne.

🔹 Attention ⚠

  • parseInt() ne prend en compte que la partie entière :
    javascript
    console.log(parseInt("10.9")); // 10 (il ignore le ".9")
  • Si la valeur contient des lettres, il s'arrête au premier caractère invalide :
    javascript
    console.log(parseInt("123abc")); // 123
console.log(parseInt("abc123")); // NaN (erreur, car ça ne commence pas par un nombre)

✅ Résumé :

✔ Convertit une chaîne en nombre entier.
✔ Évite les erreurs de comparaison ("1" !== 1).
✔ Utile pour récupérer des données du HTML comme ton data-question.

Si tu veux convertir en nombre à virgule, utilise parseFloat() à la place. 😉

javascript

javascript

✔ Dans ton code, this représente le verset cliqué.
✔ Il permet d'appliquer style.backgroundColor au bon élément.
✔ Dans une fonction normale (function()), this est l'élément qui appelle la fonction.
✔ Dans une fonction fléchée (()=>{}), this ne fonctionne pas pareil (à éviter pour les événements DOM).

🔹 === en JavaScript : C'est quoi et comment ça marche ?

L'opérateur === est un comparateur strict en JavaScript. Il vérifie à la fois la valeur et le type des deux éléments comparés.

🔹 Différence entre === et ==

OpérateurVérifie la valeur ?Vérifie le type ?Exemple avec "1" === 1"
== (égalité faible)✅ Oui❌ Nontrue (car "1" est converti en 1)
=== (égalité stricte)✅ Oui✅ Ouifalse (car "1" est une chaîne et 1 est un nombre)

🔹 Exemples concrets

✅ Avec === (égalité stricte)

javascript
console.log(5 === 5);        // true (les deux sont des nombres identiques)
console.log("5" === 5);      // false (un est une chaîne, l'autre un nombre)
console.log(true === 1);     // false (true est un booléen, 1 est un nombre)
console.log(null === undefined); // false (ce sont deux types différents)

⚠️ Avec == (égalité faible)

javascript
console.log(5 == "5");      // true (JavaScript convertit "5" en 5 avant de comparer)
console.log(true == 1);     // true (true est converti en 1)
console.log(null == undefined); // true (ils sont considérés comme équivalents)

🔹 Pourquoi utiliser === plutôt que == ?

✅ === est plus sécurisé car il empêche les conversions automatiques de JavaScript, qui peuvent parfois donner des résultats inattendus.

💡 Exemple :

javascript
const userInput = "10"; // Une chaîne
const number = 10;       // Un nombre

console.log(userInput == number);  // true (JavaScript convertit "10" en 10)
console.log(userInput === number); // false (les types sont différents)

👉 Moralité : Toujours utiliser === sauf si tu es sûr de vouloir une conversion automatique.

🔹 Exemple avec ton code

Tu utilises === ici :

javascript
if (verseQuestion === currentIndex + 1) {
    this.style.backgroundColor = "green";
} else {
    this.style.backgroundColor = "red";

📌 Ce que fait === ici :

  • verseQuestion est un nombre récupéré depuis data-question (parseInt assure que c'est un nombre).
  • currentIndex + 1 est aussi un nombre.
  • Avec ===, on s'assure que les deux sont bien identiques en valeur et en type avant de changer la couleur.

✅ Résumé

✔ === compare valeur + type → Plus sûr.
✔ == compare seulement la valeur → Moins précis, car JavaScript convertit automatiquement les types.
✔ Toujours préférer === pour éviter des bugs inattendus.
